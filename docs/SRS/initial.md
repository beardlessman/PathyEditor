# Техническое задание (ТЗ)

## Разработка клиентского веб-приложения для визуализации GPX-треков

---

### 1. Общие сведения

- **Стек технологий:** React 18+, MobX 6+ (State Management), TypeScript, Leaflet и React-Leaflet, Tailwind CSS (для стилизации).
- **Тип приложения:** Client-side SPA (без бэкенда, без авторизации, без персистентного хранения в LocalStorage/IndexedDB).
- **Цель:** Создание производительного веб-инструмента для импорта, интерактивного просмотра, анализа и последующего экспорта спутниковых треков в формате GPX.

---

### 2. Функциональные требования

#### 2.1. Модуль импорта (Загрузка файла)
- **Интерфейс:** Зона Drag-and-Drop и кнопка выбора файла через системный проводник.
- **Валидация:** Проверка расширения `.gpx` и валидация структуры XML на клиенте перед парсингом.
- **Парсинг:** Извлечение из XML-структуры тегов `<trkpt>` (широта, долгота), а также метаданных (время `<time>`, высота `<ele>`, при наличии).
- **Обработка ошибок:** Вывод тоста или текстового уведомления при загрузке повреждённого или некорректного файла.

#### 2.2. Картографический модуль (OSM & Leaflet)
- **Картооснова:** Использование бесплатных тайлов OpenStreetMap.
- **Рендеринг маршрута:** Отображение трека в виде полилинии (Polyline).
- **Поведение камеры:** Автоматическое центрирование и масштабирование карты (`fitBounds`) по крайним координатам трека сразу после завершения импорта.
- **Маркеры:** Установка уникальных маркеров для начальной точки (Старт) и конечной (Финиш).
- **Интерактивность:** Подсказки (Popup/Tooltip) над треком при клике или наведении, показывающие локальное время точки и высоту.

#### 2.3. Панель метрик и аналитики
- **Статистика трека:** Вычисление на клиенте и вывод следующих метрик:
  - Общая длина маршрута (в километрах).
  - Общее время прохождения маршрута.
  - Перепад высот: максимальная, минимальная высота, а также суммарный набор (Elevation Gain) и сброс (Elevation Loss).
- **График профиля высот:** Рендеринг интерактивного линейного графика (высота/дистанция).
- **Синхронизация:** При наведении на точку графика на карте должен отображаться временный маркер, показывающий, где именно на местности находится эта точка трека.

#### 2.4. Модуль экспорта (Скачивание файла)
- **Выгрузка:** Генерация новой XML-структуры GPX из текущего состояния данных в памяти приложения и скачивание в виде файла.
- **Именование:** Автоматическое формирование имени файла по маске `route_[YYYY-MM-DD_HHMM].gpx`.

---

### 3. Архитектура и задел под редактирование (MobX)

- **Масштабируемость:** Структура данных должна быть спроектирована с учётом будущего внедрения функций редактирования (удаление точек, разделение трека, добавление путевых точек WPT).
- **Стейт-менеджмент:**
  - **RootStore:** Главный класс для связи контекстов (на случай добавления UI-сторов, сторов настроек карты и т.д.).
  - **TrackStore:** Стор, инкапсулирующий логику трека. Массив точек маршрута должен храниться в виде плоского и мутируемого массива объектов (например, `TrackPoint[]`), чтобы в будущем операции вроде `.splice()` для удаления точек автоматически вызывали перерисовку карты.
  - **Вычисляемые свойства (`@computed` / `computed`):** Расчёт длины трека и перепада высот должен выполняться автоматически через MobX-деривативы при изменении базового массива точек.

---

### 4. Интерфейс и UX

- **Компоновка:** Двухпанельный интерфейс. Левая боковая панель — для загрузки, вывода метрик, графиков и кнопки экспорта. Правая часть — карта Leaflet на 100% оставшейся ширины и высоты экрана.
- **Адаптивность:** Скрытие боковой панели в "гамбургер-меню" или нижнюю шторку на мобильных устройствах.

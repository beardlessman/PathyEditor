# Фильтр остановок («Удаление клубков») — SRS

## 1. Назначение модуля

Удаление хаотичных скоплений точек («клубков»), возникающих из-за падения точности GPS в периоды длительных остановок пользователя (отдых, остановка, заход в магазин и т. д.). Задача фильтра — преобразовать облако ложных координат в одну корректную точку, сохранив удобочитаемую траекторию маршрута.

---

## 2. Входные данные и ограничения

**Формат входных данных:**  
Массив объектов `TrackPoint` с обязательными свойствами:

```typescript
interface TrackPoint {
  lat: number;
  lng: number;
  time: string; // Строка в формате ISO или timestamp — критично для определения длительности остановки
  ele?: number; // Высота (опционально)
}
```
> ⚠️ **Ограничение:**  
Если импортированный GPX-файл не содержит временных меток (`<time>`), фильтр **не применяется**. В интерфейсе возле имени файла отображается предупреждение:  
`Фильтр остановок недоступен: отсутствуют метки времени`.

---

## 3. Алгоритм фильтрации (Dwell Time / Stationary Filter)

1. **Инициализация:**  
   Выбирается базовая точка \(P_{start}\). Обход по последующим точкам \(P_{i}\) осуществляется в хронологическом порядке.

2. **Проверка радиуса (Spatial Check):**  
   Для каждой следующей точки \(P_{i}\) вычисляется расстояние до \(P_{start}\) (гаверсинус, с учётом кривизны Земли). Если расстояние ≤ `Radius Threshold`, точка включается в потенциальный кластер остановки.

3. **Проверка времени (Temporal Check):**  
   Как только очередная точка выходит за радиус, рассчитывается \(\Delta T = Time(P_{i-1}) - Time(P_{start})\).

4. **Удаление «клубка»:**
   - Если \(\Delta T \geq Time\ Threshold\) **и** в кластере >1 точки, все точки кластера удаляются и заменяются _одной_ усреднённой:
     - Координаты: среднее арифметическое lat/lng
     - Высота: среднее `ele` (если есть во всех точках)
     - Время: время первой точки кластера
   - Если \(\Delta T < Time\ Threshold\), рассматривается как нормальное движение — точки сохраняются без изменений.

5. **Сдвиг окна:**  
   Новая базовая точка — \(P_{i}\). Цикл повторяется до конца массива.

## 4. Интеграция в архитектуру (MobX & Pipeline)

- **Положение в конвейере:**  
  Фильтр остановок всегда применяется **первым**:
  ```
  [rawPoints] → 1. Фильтр остановок (Клубки) → 2. Фильтр шума → 3. Сжатие (RDP) → 4. Геометрическое сглаживание (Чайкин) → [points]
  ```
  > **Важно:** Если применить RDP/сглаживание раньше, форма «клубка» исказится, и расчёт по времени станет некорректным.

- **Настройки фильтра в MobX-store (`FilterStore`):**
  - `stopFilterEnabled: boolean` (по умолчанию `true`)
  - `stopRadius: number` (радиус в метрах, диапазон 5–30, шаг 5, дефолт 15)
  - `stopDuration: number` (порог времени в секундах, диапазон 60–600, шаг 60, дефолт 180)

---

## 5. Требования к интерфейсу (UI/UX)

**Боковая панель управления** содержит самостоятельный блок фильтра:
- Заголовок: _«Фильтр остановок (Удаление клубков)»_ с тумблером (включение/выключение)
- Слайдер 1: **Радиус** (`stopRadius`) — отображается значение (например, «15 метров»)
- Слайдер 2: **Время** (`stopDuration`) — отображается как человеко-понятный формат (например, «3 минуты»)
- **Реактивность:**  
  При отпускании любого ползунка (или через `debounce`, 150 мс) карта и метрики обновляются, количество итоговых точек пересчитывается сразу.

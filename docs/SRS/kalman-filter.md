# Фильтр Калмана для сглаживания GPS-треков — SRS

## 1. Назначение

Включить одномерный/двумерный фильтр Калмана в конвейер обработки GPX-треков для устранения эффекта «дрожания» (GPS jitter) и высокочастотного шума — особенно на низких скоростях, с сохранением инерции движения.

---

## 2. Пользовательский интерфейс (UI/UX)

В левую панель фильтров (**PathEditor**) добавить новый контрол-фильтр:

- **Чекбокс активации:** Фильтр Калмана
- **Слайдер "Погрешность измерения" (Measurement Noise, \(R\)):**
  - Диапазон: 1–50 метров
  - Шаг: 1 метр
  - Значение по умолчанию: 5–10 метров (типичная погрешность GPS)
- **Слайдер "Динамика движения" (Process Noise, \(Q\)):**
  - Определяет, насколько быстро фильтр откликается на реальные изменения направления
  - Диапазон: 0.1–5.0
  - Шаг: 0.1
  - Значение по умолчанию: 1.0

## 3. Математическая модель и алгоритм

В связи с неравномерным шагом по времени (\(\Delta t\)) в реальных GPX-треках применяется упрощённая кинематическая модель ("Linear Kalman Filter") для двух независимых координат (широта/долгота или X/Y после проекции).

**Состояние для каждой координаты:**  
Вектор \([x, v]^T\), где \(x\) — координата, \(v\) — скорость.

### Этапы фильтрации для каждой точки \(k\):

1. **Расчет \(\Delta t\):**  
   Время между точками \(k\) и \(k-1\) (в секундах).

2. **Экстраполяция (Prediction):**  
   - \(x_{pred} = x_{prev} + v_{prev} \cdot \Delta t\)
   - \(P_{pred} = P_{prev} + Q \cdot \Delta t\)

3. **Коррекция (Correction):**  
   - Остаток/инновация: \(y = z_k - x_{pred}\), где \(z_{k}\) — координата из GPX
   - Коэффициент Калмана: \(K = \frac{P_{pred}}{P_{pred} + R}\)
   - \(x_{new} = x_{pred} + K \cdot y\)
   - \(P_{new} = (1 - K) \cdot P_{pred}\)
   - \(v_{new} = \frac{x_{new} - x_{prev}}{\Delta t}\)

## 4. Архитектура и интеграция

### 4.1. Слой данных (**MobX Store**)

- В состояние трека добавить наблюдаемые переменные:
  - `kalmanEnabled: boolean`
  - `kalmanR: number`
  - `kalmanQ: number`
- Фильтр Калмана вызывать в общем пайплайне обработки координат.
- **Важный порядок в цепочке:**  
  Фильтр Калмана применять _до_ RDP (Douglas–Peucker), т.к. RDP удаляет точки, важные для расчёта скоростей в Калмане.

## 5. Критерии приемки

- **Производительность:** Фильтрация трека из 10 000 точек < 50 мс, отсутствует задержка интерфейса
- **Визуальный эффект:**  
  - На карте (Leaflet) при сильном зуме исчезают «лучи» в облаках точек, вызванные стоячими "GPS-клубками"
- **Сохранение геометрии:**  
  - На прямых участках трек сглажен, на резких поворотах (90˚) не происходит чрезмерное закругление (при корректном \(Q\))
